기본 자바스크립트에 강타입의 장점과 컴파일 타임의 오류를 잡아낼 수 있는 등의 장점을 가진 Typescript에 대해 알아보고 이를 활용한 웹 Backend 페이지를 구현해보고자 한다.

Typescript와 ES6+의 특성을 충분히 활용한 백엔드 프레임워크로는 NestJS가 있었고 이를 활용하여 RESTful API를 구현하고자 한다.

우선적인 목표는 NestJS CLI를 통해 프로젝트를 생성하고 TypeORM을 활용하여 DB와 연결하는 작업을 진행하고, Real World 프로젝트의 NestJS 버전 코드를 직접보면서 따라하고자 한다.

Nest CLI를 통한 프로젝트 생성

> nest new project

패키지 매니져는 yarn을 선택하여 프로젝트를 생성하였다.

project를 생성한 이후 DB와 연결을 위해 TypeORM이 DB와 연결하는데 필요한 DB 접속 정보를 ormconfig.json을 프로젝트 root 폴더에 생성하였다. 이러한 DB 접속 정보는 VCS로 변경이력을 관리할 필요도 없고 공개된 형태로 관리되어서는 보안 상에 큰 위험을 초래할 수 있으므로 .gitignore 파일에 해당 정보를 포함해야 한다.

이후에는 실제 DB 연결 라이브러리와 ORM을 설치하였다.

> yarn add @nestjs/typeorm typeorm mysql

nodemon 설치 및 설정

개발 환경에서 코드의 변화를 감지하고 이를 자동적으로 현재 구동 중인 서버에 적용하는 대표적인 라이브러리로 nodemon이 있다. 이러한 nodemon을 개발 의존성으로 설치하고 설정 파일을 설정해 보았다.

> yarn add -D nodemon

이후 프로젝트 root 폴더에 nodemon.json 파일을 생성하여 nodemon 파일을 설정한다.

src 폴더 내의 변경을 감지하되, test 파일을 제외하며 실행시 index 파일을 실행한다는 의미이다.

또한 nodemon을 실행시키기 위해 package.json의 scripts 항목에 명령어를 추가한다.

이후에는 terminal에서 다음의 명령어를 통해 nodemon을 실행할 수 있다.

> yarn start:watch

nestJS는 appModule이라는 핵심 모듈에 각 기능에 해당하는 모듈들을 포함해서 넣는 것이 가능하다. 따라서 유저 정보를 위한 모듈인 user 관련 모듈을 우선적으로 구현하기로 했다.

nestJS 각 모듈의 구성 요소들을 직접 구현할 수도 있지만 친절하게도 cli를 통해서도 해당 요소들의 뼈대를 구성할 수 있다. 따라서 cli를 통해서 해당 요소들을 만들어 보았다.
