profileModule 완성

profileModule의 경우 User 간의 following과 관련된 정보들로 구성되어 있어, 비교적 간단한 구조로 이루어져 있다.

차례대로 Controller, Service를 완성하고 ProfileRO를 위한 Interface, Module을 완성하면 된다.

Controller의 경우 User와 마찬가지로 Service를 주입받는 것 외에는 크게 다르지 않으며 Controller의 Decorator 인자로 ‘profiles’를 받아 profiles를 공통의 url로 가지는 것 외에는 동일한 구조이다.

Service에서는 특정 유저의 profile 보기, 특정 유저와의 follow, unfollow 관계를 로직으로 표현하였다.또한 차이점이 있다면, 생성자에서 UserEntity와 FollowsEntity 두 개의 Entity를 주입받았다는 점이다.

전반적인 모듈 구성은 크게 다르지 않으나 몇 가지 살펴보자면, imports의 TypeOrmModule에서 Entity가 여러 개일 경우, 해당 Entity들을 배열의 형태로 import하면 동시에 여러 개의 entity를 import 할 수 있다.
또한 기존의 UserModule에서는 exports를 통해 UserService를 export하여 해당 로직을 다른 모듈에서도 사용할 수 있었다.

다음에는 article과 관련된 로직을 완성해 볼 예정이다.
