UserModule 완성

회원가입, 로그인 유저 정보 확인 두 개의 기능이 구현되었고 나머지 3개의 기능으로 회원 정보 수정, 로그인, 회원 삭제 기능을 구현하여 UserModule을 마무리 할 예정이다.

로그인과 관련된 UserDto 정보를 DTO로 선언하여 해당 객체로 접근하고 user를 검색하여 검색 결과가 없으면 에러를 그게 아닐 경우 token을 새로 생성하여 user를 반환한다.

로그인 된 유저가 DB에 접근할 떄, email과 password로 검색을 하되, password는 암호화 방식으로 변경되어 저장되므로 암호화 후에 저장하였다.

업데이트 및 삭제의 경우, 특정 사용자에 대해 업데이트 삭제가 이루어지게 된다. 삭제의 경우는 url 내 parameter를 통해서 삭제할 계정 정보를 가져온다.

마지막으로 현재 백엔드에 대한 API 문서화를 자동으로 생성해주는 Swagger를 적용해보도록 하곘다.
우선 프로젝트 전체에 Swagger에 대한 설정은 지정해야 한다.

우선 Swagger에서 문서 생성시 사용할 옵션에 대해 builder 패턴을 통해 옵션을 지정한다. 해당 옵션으로 모듈을 생성하고 url 경로와 함께 Nest와 연결하면 간편하게 설정이 완료된다.

마지막으로 decorator를 이용하여 User의 Controller에 두 개의 설정을 추가해주면 Swagger 문서 생성이 자동으로 완성된다. 완성된 swagger 페이지는 main.ts에서 선언했던대로 /docs url을 입력하면 확인 할 수 있다.

이후에는 Profile 관련 모듈을 생성할 예정이다.
