backend 오류 수정

react 관련 readworld 예제를 가지고 RESTful API를 점검해본 결과 일부 Article API의 문제를 수정하였다.

feed가 :slug 밑에 있는 경우 :slug가 feed 라우팅까지 가로채므로 feed를 위로 올렸다.

일단 크게 세 부분이 수정되었다. 전체 게시글을 불러올 떄, query.author를 통해 author 정보를 올바르게 불러오도록 하였다.

두 번째와 세 번째는 결과적으로 author 정보가 게시글에 실리게 할 수 있는 조치이다. 하지만 그 방식이 다른데, 전자의 경우는 querybuilder에서 query를 생성할 때, author에 해당하는 테이블을 조인하여 불러오도록 하는 것이고, 후자의 경우는 findOne 명령을 수행할 때, relations에 다른 테이블과 관계가 있는 column인 부분을 입력하면 검색 시에 author에 해당하는 정보도 같이 검색을 수행하게 된다.

frontend 탐색

기존의 realworld 예제에 있는 react를 활용한 프론트 예제는 redux를 사용하는 등 예제는 괜찮았으나 비교적 옛날에 만들어진 예제라 그런지 Hooks 기반으로 작성된 것이 아닌 class 기반으로 작성되었으며, typescript가 적용되어 있지 않아 현재 프로젝트에 채용하기에는 아쉬운 점이 있었다. 따라서 typescript를 활용한 예제에 redux와 redux-saga를 사용하는 예제를 찾아보았지만 마땅한 것이 없어서 프로젝트 진행이 어려웠다.

그러던 도중, react와 typescript를 가지고 작성된 realworld 예제를 찾게 되었다. redux와 같은 모듈을 사용하지 않고 react의 Context API를 이용하여 redux의 기능을 구현했다. redux 역시 context API로 작성되었다 알고 있었기 때문에 해당 예제를 채택하게 되었다.

RealWorld(React + Typescript)

frontend 프로젝트 생성

기본적인 프론트엔드 프로젝트는 create-react-app을 기반으로 프로젝트를 생성한다. yarn의 경우 typescript를 기반으로 프로젝트를 생성할 때는 다음과 같은 명령어를 사용한다.

> yarn create react-app 프로젝트명 –typescript

이와 같이 실행하면 typescript 기반의 tsx로 프로젝트가 생성된다. 이를 기반으로 realworld 예제를 사용해 보았다.

전체적인 프로젝트 구성은 위와 같다. API는 axios 설정 등 API와 관련 설정 파일들이 모여있는 것이고, components는 컴포넌트 파일들을 모아놓는 곳이다. context 폴더는 contextAPI와 관련된 경로, reducers 폴더는 redux의 reducer와 같이 action, reducer나 type 등을 가지고 있다. 마지막으로 types는 interface와 관련된 내용이 포함되어 있다.

현재는 Header, Home 컴포넌트까지만 구현을 완성하였다.
