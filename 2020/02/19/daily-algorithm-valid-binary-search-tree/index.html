이진 검색 트리(Binary Search Tree)

이진 검색 트리는 검색에 있어 속도를 빠르게 하기 위해 고유한 키값을 가지고 부모 노드를 기준으로 작은 값은 자식 노드의 왼쪽에, 큰 값은 자식 노드의 오른쪽에 배치한다.
이러한 BST의 원리를 통해 데이터베이스에서 index를 생성하고 검색 속도를 개선하게 된다.

Problem

주어진 입력이 올바른 이진 트리인지 확인하여 true, false를 판별한다. LeetCode 문제 출처

Input: [2, 1, 3] (Binary Tree), Output: true
Input: [5, 1, 4, null, null, 3, 6], Output: false

solving

재귀를 통해 입력되는 값들을 분할하여 처리한다.

O(n)의 복잡도를 갖는 무난한 풀이 방법이다.

소감

처음에 입력 값이 배열처럼 표시되어 있어 배열로 계산하느라 생각보다 시간이 오래 걸렸다.
하지만 기본적인 BST의 기본을 다룰 수 있는 좋은 문제였다고 생각한다.
시간 여유가 있다면 배열 입력도 해결할 수 있는 알고리즘으로 짜는 거도 좋은 것 같다.
